# 如何恰当让 ai 不过量使用 token

这套是给你这种做：
* 后端架构
* Docker / 多服务器
* 数据库迁移
* 长设计对话

🎯 核心原则（先理解这个）
Claude 会卡你的原因只有三个：
1. 单轮输出太大
2. 会话上下文太长
3. 连续高强度调用
所以解决思路是：
> ✅ 把“长对话”变成“分段项目”
> ✅ 把“持续上下文”变成“本地文件”
> ✅ 把“模型记忆”转移到 Markdown

🚀 第一部分：项目结构（关键）

建立一个固定目录：

```bash
~/AI/
  claude/
    sessions/
    summaries/
    architecture/
    migrations/
    notes/
```
以后：
* 设计架构 → architecture/
* 迁移脚本 → migrations/
* 会话摘要 → summaries/

🧠 第二部分：分段式对话法（极重要）
不要这样问：
> 给我一个完整生产级多服务器部署方案

会炸。
改成这样：
### 第一步
```text
只设计整体系统模块图，不要展开代码
```
保存。

---

### 第二步

```text
基于刚才的模块设计，只实现 API 网关部分
```

保存。

---

### 第三步

```text
只设计数据库迁移策略
```

这样：

* 每次输出变小
* token 消耗稳定
* 不会 5 小时爆掉

---

🧩 第三部分：上下文转移法（核心技巧）

当对话变长时：

不要继续聊。

让 Claude：

```text
请将当前阶段内容压缩为 800 token 技术摘要
只保留：
- 架构决策
- 关键参数
- 约束条件
```
保存为：
```
PROJECT_STAGE_1_SUMMARY.md
```
然后：
```bash
claude --new
```
再输入：
```text
基于 PROJECT_STAGE_1_SUMMARY.md 继续实现缓存层
```
这样你永远不会爆。

🛠 第四部分：限流控制技巧

### 1️⃣ 控制输出长度

在问题后加：

```
限制在 500 token 内
```

或：

```
不要给完整代码，只给核心结构
```

---

### 2️⃣ 避免让它重复历史

不要说：

> 基于我们刚才 20000 字的讨论

要改成：

> 基于下面摘要

---

# 🔥 第五部分：5 小时限制破解思路

理解一个事实：

Claude 的限制是：

> 过去 5 小时内的使用量

所以策略是：

### 工作节奏建议

* 每 60–90 分钟暂停一次
* 做本地整理
* 不连续猛刷

---

# 📊 第六部分：你的最佳工作节奏（推荐）

你适合这种模式：

### 1️⃣ 设计阶段（30 分钟）

连续问架构问题。

---

### 2️⃣ 冷却阶段（20 分钟）

整理本地 markdown。

---

### 3️⃣ 再开新会话继续

这样几乎不会被限。

---

# 💡 高级技巧（很多人不知道）

### Claude 真正耗 token 的是：

* 大量上下文回读
* 大段代码生成
* 连续大输出

不是简单对话次数。

所以：

> 把“记忆”从 Claude 转移到文件
> 才是终极解法

---

# 🎯 如果 5 小时真的被锁了怎么办？

两种方案：

### 方案 A（最干净）

等 1–2 小时
额度自然恢复。

---

### 方案 B（临时过渡）

切换到 API key 模式继续（你之前问过）

CLI 支持：

```
ANTHROPIC_API_KEY=xxx claude
```

继续干活。

# 🏗 给你一个真正企业级工作流总结

1. 每阶段结束就导出 md
2. 每 10000 字就压缩摘要
3. 不做 3 小时连续长对话
4. 所有关键决策本地化
5. 用 --new 分阶段
