
# elastic/elasticsearch 和 aws/opensearch 那个三方库 更优秀，坑更少？

背景

OpenSearch 是 AWS 在 2021 年从 Elasticsearch 7.10 fork 出来的。两个客户端库 API 几乎一样，因为 OpenSearch 客户端就是从
elastic 客户端 fork 的。

已知的坑

@elastic/elasticsearch
- v7 → v8 有破坏性变更（body 参数被拍平，升级要改代码）
- v8 强制要求连接 Elastic 官方发行版，连 OpenSearch 会警告
- 高版本客户端连低版本 ES 可能有兼容问题

@opensearch-project/opensearch
- 类型定义不如 elastic 完整，部分 API 要用 any
- 文档质量差，很多场景只能去看 elastic 的文档然后自己适配
- 新特性（如 k-NN、向量搜索）的客户端支持滞后
- 社区小，遇到问题 StackOverflow 上答案少

你的项目应该选哪个

继续用 @elastic/elasticsearch，理由：

什么时候该换 OpenSearch

只有一种情况：你把搜索引擎从 Elasticsearch 换成了 AWS OpenSearch Service。那时候必须用 OpenSearch 客户端，因为 elastic
客户端 v8 会主动拒绝连接非 Elastic 发行版。

除此之外，没有理由换。两个库本质上是同一套代码的两个分支，elastic 那边迭代更快、类型更好、文档更全。
