## 2. 核心服务清单

| 服务 | 职责 | 数据存储 | 通信方式 |
|------|------|----------|----------|
| **API Gateway** | 认证、限流、路由分发 | Redis(令牌桶) | HTTP入口 |
| **用户服务** | 注册/登录/权限/Profile | PG + Redis会话 | REST |
| **商品服务** | 商品CRUD、分类、SKU | PG + Redis热点缓存 | REST |
| **库存服务** | 库存扣减(Redis预扣+PG落盘) | PG + Redis | MQ + REST |
| **订单服务** | 下单、状态机、履约 | PG分表(按用户ID) | MQ + REST |
| **支付服务** | 支付对接、回调、对账 | PG独立库 | REST + Webhook |
| **搜索服务** | 全文检索、筛选、推荐 | Elasticsearch | REST |
| **通知服务** | 短信/邮件/站内信/推送 | MQ消费 | MQ |
| **营销服务** | 优惠券/秒杀/满减/活动 | Redis + PG | REST + MQ |

## 3. 服务间关系

```
用户 ──→ API Gateway ──→ 用户服务
                    ├──→ 商品服务 ──→ 搜索服务
                    ├──→ 订单服务 ──MQ──→ 库存服务
                    │         └──MQ──→ 通知服务
                    ├──→ 支付服务 ──Webhook──→ 订单服务
                    └──→ 营销服务
```

**调用规则**:
- 同步调用: Gateway → 各服务 (REST/HTTP)
- 异步解耦: 订单 → 库存/通知 (MQ)
- 回调通知: 支付平台 → 支付服务 → 订单服务

## 4. 数据层设计

### PostgreSQL
- **主从复制**: 1主 2从, 读写分离
- **分库**: 用户库 / 商品库 / 订单库 / 支付库
- **分表**: 订单表按 `user_id % 64` 水平分表

### Redis Cluster
- **会话管理**: JWT + Redis 黑名单
- **热点缓存**: 商品详情 TTL 5min + 随机偏移(防雪崩)
- **库存预扣**: `DECR` 原子操作, 异步落盘PG
- **分布式锁**: Redlock, 用于秒杀/库存
- **布隆过滤器**: 防缓存穿透

### 消息队列 (NATS)
- 订单创建 → 库存扣减
- 支付成功 → 订单状态更新 → 通知发送
- 商品变更 → 搜索索引更新

## 5. 部署架构 (Docker)

```
┌─ Docker Swarm / K8s Cluster ─────────────────┐
│                                               │
│  ┌─ Node 1 ──────┐  ┌─ Node 2 ──────┐       │
│  │ api-gw x2     │  │ api-gw x2     │       │
│  │ user-svc x2   │  │ order-svc x2  │       │
│  │ product-svc x2│  │ pay-svc x2    │       │
│  └────────────────┘  └────────────────┘       │
│                                               │
│  ┌─ Node 3 ──────┐  ┌─ Data Node ───┐       │
│  │ stock-svc x2  │  │ PG主          │       │
│  │ search-svc x2 │  │ Redis x3      │       │
│  │ notify-svc x2 │  │ NATS x3       │       │
│  └────────────────┘  └────────────────┘       │
└───────────────────────────────────────────────┘
```

**容器编排要点**:
- 每服务独立 Dockerfile + 镜像
- `docker-compose.yml` 开发环境
- K8s HPA 自动扩缩容 (CPU > 70% 触发)
- 健康检查: `/health` 端点, 10s 间隔
