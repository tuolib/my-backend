阶段四：核心服务实现（用户/商品/订单）

⏺ 核心服务总览（Bun + Hono + PostgreSQL + Redis）

1. 模块边界

┌──────────┬──────────────────────────────────────────────┬────────────────────────────┐
│   服务   │                     职责                     │         主要存储           │
├──────────┼──────────────────────────────────────────────┼────────────────────────────┤
│ 用户服务 │ 注册/登录/JWT签发、资料维护、账号状态管理     │ PG(users) + Redis(session) │
├──────────┼──────────────────────────────────────────────┼────────────────────────────┤
│ 商品服务 │ 商品/SPU/SKU 查询与管理、上下架、价格与库存展示 │ PG(products/skus) + Redis  │
├──────────┼──────────────────────────────────────────────┼────────────────────────────┤
│ 订单服务 │ 下单、订单状态流转、订单查询、库存预扣协同      │ PG(orders_00~63) + Redis   │
└──────────┴──────────────────────────────────────────────┴────────────────────────────┘

2. 服务间调用关系

Client → API Gateway
├─ /api/v1/users/*   → 用户服务
├─ /api/v1/products/*→ 商品服务
└─ /api/v1/orders/*  → 订单服务

订单关键链路：
下单请求 → 订单服务校验参数与幂等 → Redis 预扣库存 → 写订单分表 → 写库存流水/Outbox → 返回订单结果

3. 核心接口域（REST）

用户服务
- POST `/api/v1/users/register`  用户注册
- POST `/api/v1/users/login`     用户登录（JWT）
- GET  `/api/v1/users/profile`   用户资料

商品服务
- GET  `/api/v1/products`        商品列表（支持分页/筛选）
- GET  `/api/v1/products/:id`    商品详情（优先缓存）
- GET  `/api/v1/products/:id/skus` SKU 列表

订单服务
- POST `/api/v1/orders`          创建订单
- GET  `/api/v1/orders/:id`      订单详情
- GET  `/api/v1/orders`          用户订单列表
- POST `/api/v1/orders/:id/cancel` 取消订单

4. 统一实现约束

- 分层结构：`route -> service -> repository`
- 统一响应：`{ code, success, message, data }`
- 参数校验：路由层完成 schema 校验，service 不信任外部输入
- 错误处理：业务错误与系统错误分离，禁止向客户端暴露堆栈
- 日志要求：关键路径记录 `requestId/userId/orderId/latency`
- 幂等要求：下单必须支持 `idempotency_key`

5. 数据一致性与事务边界

- 用户服务：注册/登录写操作在单库事务内完成
- 商品服务：商品写入后主动失效 `product:{id}` 缓存
- 订单服务：
  - 订单写入与库存流水写入在同一 DB 事务内
  - Redis 预扣成功但 DB 落盘失败时，依赖 Outbox 重试/补偿
  - 分表路由固定规则：`user_id % 64 -> orders_00~63`

6. 性能与稳定性基线

- 读路径优先缓存：商品详情命中 Redis，未命中回源 PG 并回填
- 写路径削峰：库存扣减先 Redis 原子操作，再异步/重试落盘
- 限流与超时：依赖网关统一限流，服务内设置 DB/Redis 超时
- 降级策略：Redis 故障 fail-open（查询降级），订单写链路优先保证可追踪

7. 阶段四交付清单

- 用户服务：注册/登录/Profile 全链路可用
- 商品服务：列表/详情/SKU 查询可用，缓存回填生效
- 订单服务：创建/查询/取消可用，分表路由正确，库存预扣链路可追踪
