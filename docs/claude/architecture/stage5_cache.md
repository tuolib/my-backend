阶段四：缓存设计

⏺ Redis 缓存方案

1. 数据过期策略

┌─────────────────────┬──────────────────┬──────────────────────────────────┐
│      Key 模式       │       TTL        │               说明               │
├─────────────────────┼──────────────────┼──────────────────────────────────┤
│ session:{uid}       │ 24h              │ JWT 会话，登出写入黑名单         │
├─────────────────────┼──────────────────┼──────────────────────────────────┤
│ product:{id}        │ 5min + rand(60s) │ 商品详情，随机偏移防雪崩         │
├─────────────────────┼──────────────────┼──────────────────────────────────┤
│ stock:{sku_id}      │ 不过期           │ 库存计数器，仅由库存服务同步重置 │
├─────────────────────┼──────────────────┼──────────────────────────────────┤
│ rate:{ip}:{uid}     │ 1s 滑窗          │ 令牌桶限流，自然过期回收         │
├─────────────────────┼──────────────────┼──────────────────────────────────┤
│ bloom:products      │ 不过期           │ 布隆过滤器，商品变更时重建       │
├─────────────────────┼──────────────────┼──────────────────────────────────┤
│ lock:stock:{sku_id} │ 3s               │ Redlock 分布式锁，自动释放       │
└─────────────────────┴──────────────────┴──────────────────────────────────┘

2. 缓存与 ES 查询结合

搜索请求 → ES 返回 id 列表
→ Redis MGET product:{id1},{id2}...
├─ 全部命中 → 组装返回
└─ 部分未命中 → PG 批量查询缺失项
→ Pipeline SET EX 回填 Redis
→ 合并返回

- ES 只存索引字段（title/category/price），不存详情
- 商品变更 → MQ → ES 增量更新 + Redis DEL 失效缓存

3. 防雪崩机制

- 随机 TTL: 基础 300s + rand(0~60s)，打散过期时间点
- singleflight: 同一 key 并发回源时，仅放行 1 个请求查 PG，其余等待结果
- 降级兜底: Redis 不可用时直接查 PG，熔断器限制回源 QPS 防止 DB 被打满
- 预热: 大促前批量 Pipeline SET 热点商品，TTL 设为活动时长
