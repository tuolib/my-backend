
阶段七：子模块详细设计

⏺ API 网关子模块详细设计

1. 接口实现（Hono 中间件链）

请求 → [requestId] → [rateLimit] → [auth] → [proxy] → 响应

中间件执行顺序:
① requestId  — 无 X-Request-ID 则生成 uuid，注入 c.set('rid')
② rateLimit  — Redis EVAL lua: 双桶检查 rate:{ip} + rate:{uid}
③ auth       — 解析 JWT → Redis GET blacklist:{jti} → 拒绝/放行
④ proxy      — 路由表匹配 → fetch(downstream) → 统一包装返回

路由表结构:
const routes = {
'/api/users/**':    { target: 'http://user-svc:3001',    timeout: 3000 },
'/api/products/**': { target: 'http://product-svc:3002',  timeout: 3000 },
'/api/orders/**':   { target: 'http://order-svc:3003',    timeout: 3000 },
'/api/payments/**': { target: 'http://pay-svc:3004',      timeout: 10000 },
}

2. 异常处理（分层捕获）

┌──────────┬─────────────────────┬───────────────────────────────────────────────────────┐
│   层级   │        异常         │                         处理                          │
├──────────┼─────────────────────┼───────────────────────────────────────────────────────┤
│ 限流层   │ 令牌耗尽            │ 返回 429 + Retry-After 头                             │
├──────────┼─────────────────────┼───────────────────────────────────────────────────────┤
│ 认证层   │ JWT 过期/黑名单     │ 返回 401，不转发                                      │
├──────────┼─────────────────────┼───────────────────────────────────────────────────────┤
│ 代理层   │ 下游超时            │ 返回 504，记录慢请求                                  │
├──────────┼─────────────────────┼───────────────────────────────────────────────────────┤
│ 代理层   │ 下游 5xx / 连接拒绝 │ 熔断计数 +1，返回 502 + 降级 JSON                     │
├──────────┼─────────────────────┼───────────────────────────────────────────────────────┤
│ 熔断器   │ 状态=OPEN           │ 直接返回 503 Service Unavailable，半开探测每 10s 一次 │
├──────────┼─────────────────────┼───────────────────────────────────────────────────────┤
│ 全局兜底 │ 未捕获异常          │ app.onError 返回 500，脱敏，不泄露堆栈                │
└──────────┴─────────────────────┴───────────────────────────────────────────────────────┘

3. 日志策略

每条请求日志（JSON 结构化）:
{
rid: "X-Request-ID",
method, path, status,
uid: "从JWT解析",
ip,
latency_ms: "响应耗时",
upstream: "下游服务名",
error?: "异常摘要（脱敏）"
}
