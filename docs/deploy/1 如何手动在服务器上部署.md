❯ 那现在服务器上执行步骤是什么？我是一个新手

⏺ 从零部署完整步骤

假设你有 5 台服务器（Ubuntu/Debian），IP 如下（替换成你的真实 IP）：

┌────────┬──────────────────────────┬──────────┐                                                                                     
│ 服务器 │           角色           │ 示例 IP  │                                    
├────────┼──────────────────────────┼──────────┤                                                                                     
│ S1     │ 数据层（PG主 + Redis主） │ 10.0.0.1 │                                                                                     
├────────┼──────────────────────────┼──────────┤                                                                                     
│ S2     │ 数据层（PG从 + Redis从） │ 10.0.0.2 │                                                                                     
├────────┼──────────────────────────┼──────────┤                                                                                     
│ S3     │ 入口层（Caddy）          │ 10.0.0.3 │
├────────┼──────────────────────────┼──────────┤
│ S4     │ 应用层（Worker）         │ 10.0.0.4 │
├────────┼──────────────────────────┼──────────┤
│ S5     │ 应用层（Worker）         │ 10.0.0.5 │
└────────┴──────────────────────────┴──────────┘

  ---
第 1 步：5 台服务器都装 Docker

每台服务器都执行：

# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker 并设为开机自启
sudo systemctl enable --now docker

# 让当前用户可以用 docker（免 sudo）
sudo usermod -aG docker $USER

# 退出重新登录，使生效
exit

重新 SSH 登录后验证：
docker --version
# 应该输出 Docker version 2x.x.x

  ---
第 2 步：把项目代码传到 S1

在你本地电脑执行：

# 方法1：如果服务器能访问 Git 仓库
# 在 S1 上执行：git clone <你的仓库地址>

# 方法2：直接打包上传
cd /Users/test/Downloads/ho
tar czf project.tar.gz --exclude=node_modules --exclude=.git .
scp project.tar.gz root@10.0.0.1:/opt/ecom/project.tar.gz

在 S1 上解压：
mkdir -p /opt/ecom && cd /opt/ecom
tar xzf project.tar.gz

  ---
第 3 步：搭建私有镜像仓库（在 S1 上）

因为 Swarm 的 Worker 节点需要拉取你的应用镜像，你需要一个镜像仓库。最简单的方式是在 S1 上跑一个：

# 在 S1 上启动一个私有 Registry
docker run -d --restart=always -p 5000:5000 --name registry registry:2

然后每台服务器都要配置信任这个私有仓库（因为它是 HTTP 不是 HTTPS）：

# 每台服务器都执行（把 10.0.0.1 换成 S1 的真实 IP）
cat > /etc/docker/daemon.json <<'EOF'
{
"insecure-registries": ["10.0.0.1:5000"]
}
EOF

sudo systemctl restart docker

  ---
第 4 步：构建并推送镜像（在 S1 上）

cd /opt/ecom

# 设置 Registry 地址（S1 的 IP）
export REGISTRY=10.0.0.1:5000
export TAG=v1.0.0

# 逐个构建并推送（共 5 个服务）
# API Gateway
docker build -t $REGISTRY/ecom-api-gateway:$TAG -f apps/api-gateway/Dockerfile .
docker push $REGISTRY/ecom-api-gateway:$TAG

# User Service
docker build -t $REGISTRY/ecom-user-service:$TAG -f services/user-service/Dockerfile .
docker push $REGISTRY/ecom-user-service:$TAG

# Product Service
docker build -t $REGISTRY/ecom-product-service:$TAG -f services/product-service/Dockerfile .
docker push $REGISTRY/ecom-product-service:$TAG

# Cart Service
docker build -t $REGISTRY/ecom-cart-service:$TAG -f services/cart-service/Dockerfile .
docker push $REGISTRY/ecom-cart-service:$TAG

# Order Service
docker build -t $REGISTRY/ecom-order-service:$TAG -f services/order-service/Dockerfile .
docker push $REGISTRY/ecom-order-service:$TAG

或者用脚本一键构建：
export REGISTRY=10.0.0.1:5000 TAG=v1.0.0
./infra/swarm/deploy.sh build

  ---
第 5 步：初始化 Swarm 集群

在 S1 上执行（第一个 Manager）：

# 初始化 Swarm（用 S1 的内网 IP）
docker swarm init --advertise-addr 10.0.0.1

执行后会输出两条命令，复制保存下来，类似：

To add a manager to this swarm, run:
docker swarm join --token SWMTKN-1-xxxxx-manager 10.0.0.1:2377

To add a worker to this swarm, run:
docker swarm join --token SWMTKN-1-xxxxx-worker 10.0.0.1:2377

  ---
第 6 步：其他节点加入集群

在 S2 上执行（Manager）：
docker swarm join --token SWMTKN-1-xxxxx-manager 10.0.0.1:2377

在 S3 上执行（Manager）：
docker swarm join --token SWMTKN-1-xxxxx-manager 10.0.0.1:2377

在 S4 上执行（Worker）：
docker swarm join --token SWMTKN-1-xxxxx-worker 10.0.0.1:2377

在 S5 上执行（Worker）：
docker swarm join --token SWMTKN-1-xxxxx-worker 10.0.0.1:2377

回到 S1 验证：
docker node ls

应该看到 5 个节点，3 个 Manager，2 个 Worker：

abc123 *     s1         Ready    Active         Leader
def456       s2         Ready    Active         Reachable
ghi789       s3         Ready    Active         Reachable
jkl012       s4         Ready    Active
mno345       s5         Ready    Active

  ---
第 7 步：设置节点标签（在 S1 上）

告诉 Swarm 每台服务器的角色：

# 把 s1、s2、s3 替换成 docker node ls 显示的 HOSTNAME
docker node update --label-add role=db-primary  s1
docker node update --label-add role=db-replica  s2
docker node update --label-add role=gateway     s3
# S4、S5 是 Worker，不需要标签

验证：
docker node inspect s1 --format '{{.Spec.Labels}}'
# 输出: map[role:db-primary]

  ---
第 8 步：创建 Secrets（在 S1 上）

# 生成安全的随机密码（也可以自己指定）
# 每个 secret 执行一次

# PostgreSQL 密码
echo -n "你的PG强密码放这里" | docker secret create postgres_password -

# JWT Access Token 密钥（至少 16 字符）
echo -n "你的jwt-access-secret放这里至少16字符" | docker secret create jwt_access_secret -

# JWT Refresh Token 密钥（至少 16 字符）
echo -n "你的jwt-refresh-secret放这里至少16字符" | docker secret create jwt_refresh_secret -

# 服务间通信密钥（至少 8 字符）
echo -n "你的internal-secret放这里" | docker secret create internal_secret -

如果想用随机生成的密码：
openssl rand -base64 32 | docker secret create postgres_password -
openssl rand -base64 48 | docker secret create jwt_access_secret -
openssl rand -base64 48 | docker secret create jwt_refresh_secret -
openssl rand -base64 32 | docker secret create internal_secret -

验证：
docker secret ls
# 应该看到 4 个 secret

  ---
第 9 步：DNS 解析（域名指向 S3）

去你的域名管理面板，添加一条 A 记录：

api.find345.site → S3 的公网 IP

同时确保 S3 的防火墙开放 80 和 443 端口。

  ---
第 10 步：部署！（在 S1 上）

cd /opt/ecom

export REGISTRY=10.0.0.1:5000
export TAG=v1.0.0

docker stack deploy -c infra/swarm/docker-stack.yml ecom

  ---

  ---
常用运维命令速查

# 查看所有服务状态
docker stack services ecom

# 查看某个服务的日志（-f 实时跟踪）
docker service logs -f ecom_api-gateway

# 更新镜像（重新构建后）
export REGISTRY=10.0.0.1:5000 TAG=v1.0.1
docker service update --image $REGISTRY/ecom-api-gateway:$TAG ecom_api-gateway

# 扩缩容（临时加副本）
docker service scale ecom_api-gateway=4

# 完整重新部署（修改了 stack 文件后）
docker stack deploy -c infra/swarm/docker-stack.yml ecom

# 删除整个 stack
docker stack rm ecom
