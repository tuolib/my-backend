架构：Docker Swarm 5 节点集群 + GitHub Actions CI/CD 自动部署

节点规划：
- S1/S2/S3：Swarm Manager（S1 为主管理节点）
- S4/S5：Swarm Worker

节点职责（通过 Swarm 标签约束）：

┌───────┬─────────────────┬───────────────────────────────────────────┐
│ 节点  │      标签       │                 运行服务                  │
├───────┼─────────────────┼───────────────────────────────────────────┤
│ S1    │ role=db-primary │ PostgreSQL 主 + Redis 主 + Redis Sentinel │
├───────┼─────────────────┼───────────────────────────────────────────┤
│ S2    │ role=db-replica │ PostgreSQL 从 + Redis 从 + Redis Sentinel │
├───────┼─────────────────┼───────────────────────────────────────────┤
│ S3    │ role=gateway    │ Nginx 反代 + Certbot SSL + Redis Sentinel │
├───────┼─────────────────┼───────────────────────────────────────────┤
│ S4/S5 │ 无              │ 5 个应用微服务（Swarm 自动调度）          │
└───────┴─────────────────┴───────────────────────────────────────────┘

网络隔离（3 层 overlay）：
- public_net：Nginx ↔ API Gateway
- service_net：API Gateway ↔ 微服务
- data_net（internal）：微服务 ↔ PG/Redis，外部不可达

SSL：Certbot 容器自动签发 Let's Encrypt 证书，12h 续签

CI/CD：GitHub Actions 构建镜像推 GHCR → SSH 到 S1 → docker stack deploy → 自动迁移 + seed

运维只需做：
1. 5 台 Vultr 服务器装 Docker，在同一 VPC
2. S1 初始化 Swarm，S2/S3 以 Manager 加入，S4/S5 以 Worker 加入
3. 按表打标签
4. S1 开放 SSH，公钥给 GitHub Actions
5. 节点间开放 2377/7946/4789，对外开放 80/443
6. DNS A 记录指向 S3/S4/S5

其余全部由 GitHub Actions 自动完成：Secrets 创建、镜像拉取、Stack 部署、滚动更新、数据库迁移。所有配置文件已在仓库
infra/swarm/ 目录中。
