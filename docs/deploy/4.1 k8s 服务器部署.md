你的服务器是哪个云厂商？这决定了用托管 K8s 还是自建集群。

1. 阿里云 ACK (推荐)                                                                                                                                                                               
   控制面免费，最适合国内业务，自动集成 SLB/EIP
2. AWS EKS                                                                                                                                                                                         
   全球化部署首选，生态最成熟
3. 腾讯云 TKE                                                                                                                                                                                      
   国内第二选择，控制面免费                                                                                                                                                                        
   ❯ 4. 自建机房 (kubeadm)                                                                                                                                                                              
   需要自己搭建控制面，运维成本最高

数据层（PostgreSQL + Redis）如何处理？

1. 保留外部独立部署 (推荐)                                                                                                                                                                         
   S1/S2 的 Patroni + Redis Sentinel 不动，K8s 只跑应用层，风险最小                                                                                                                                
   ❯ 2. 全部迁入 K8s ✔                                                                                                                                                                                  
   使用 CloudNativePG + Redis Operator，运维统一但改动大、风险高
3. 用云托管数据库                                                                                                                                                                                  
   RDS + ElastiCache 等托管服务，最省心但成本最高
