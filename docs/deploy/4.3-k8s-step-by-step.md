# K8s 部署一步一步操作指南

## 第 1 步：服务器先准备好 K8s 集群

在动 GitHub Actions 之前，先确保集群能跑：

```bash
# 在 5 台服务器上依次执行（已有脚本）
# 所有节点
./infra/k8s/cluster-setup/01-prerequisites.sh

# S1
./infra/k8s/cluster-setup/02-init-control-plane.sh

# S2, S3
./infra/k8s/cluster-setup/03-join-control-plane.sh

# S4, S5
./infra/k8s/cluster-setup/04-join-workers.sh

# S1
./infra/k8s/cluster-setup/05-install-cni.sh
./infra/k8s/cluster-setup/06-install-operators.sh
```

## 第 2 步：在 S1 上安装 Helm

```bash
# S1 上执行
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
helm version   # 验证
```

## 第 3 步：手动部署一次，确认能跑

```bash
# 在 S1 上手动跑一次，验证 Helm Chart 没问题
cd /opt/ecom/infra/k8s
REGISTRY=ghcr.io/<你的用户名> ./deploy.sh full
```

## 第 4 步：配置 GitHub Secrets

到你的 GitHub 仓库 → Settings → Secrets and variables → Actions，添加：

| Secret 名 | 值 | 说明 |
|-----------|------|------|
| `K8S_SSH_HOST` | S1 的 IP | 控制平面节点 |
| `K8S_SSH_USER` | `root` 或部署用户 | SSH 用户 |
| `K8S_SSH_KEY` | S1 的 SSH 私钥 | `cat ~/.ssh/id_ed25519` |
| `K8S_POSTGRES_PASSWORD` | 你的 PG 密码 | Helm 传入 |
| `K8S_REPLICATION_PASSWORD` | 复制密码 | Helm 传入 |
| `K8S_JWT_ACCESS_SECRET` | JWT 密钥 | Helm 传入 |
| `K8S_JWT_REFRESH_SECRET` | JWT 刷新密钥 | Helm 传入 |
| `K8S_INTERNAL_SECRET` | 内部通信密钥 | Helm 传入 |

## 第 5 步：推送代码触发自动部署

GitHub Actions workflow 文件已就绪（`.github/workflows/deploy.yml`），push 到 main 即自动触发：

```
类型检查 → 并行构建 5 个镜像 → SSH 到 S1 → Helm 部署 → 健康检查
```

## 第 6 步（可选）：手动触发部署

在 GitHub 仓库 → Actions → "Build & Deploy (K8s)" → Run workflow，可指定自定义镜像 tag。

## 新手注意事项

1. **先手动跑通再自动化** — 第 3 步很重要，别跳过
2. **Secrets 别泄露** — 永远不要把密码写在代码里，都放 GitHub Secrets
3. **GHCR 镜像权限** — 如果仓库是 private，K8s 节点需要 pull secret 才能拉镜像（workflow 里已包含）
4. **回滚很简单** — 出问题了在 S1 上执行 `helm rollback ecom <版本号>`
5. **查看状态** — 在 S1 上执行 `kubectl get pods -n ecom` 查看所有 Pod 状态
