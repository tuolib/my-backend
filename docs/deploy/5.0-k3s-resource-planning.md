# K3s 5 节点集群资源规划

## 节点角色建议（5 台服务器）

| 角色 | 数量 | 说明 |
|------|------|------|
| Server (control plane) | 3 | 高可用 etcd 集群，奇数节点 |
| Agent (worker) | 2 | 运行业务负载 |

---

## K3s 系统组件资源需求

### Server 节点（每台）

| 组件 | CPU（最低） | RAM（最低） | 说明 |
|------|------------|------------|------|
| k3s server 进程 | 200m | 512 Mi | 含 API Server + Controller Manager + Scheduler |
| 嵌入式 etcd | 200m | 256 Mi | 3 节点 HA 模式 |
| containerd | 100m | 128 Mi | 容器运行时 |
| CoreDNS | 100m | 70 Mi | 集群 DNS |
| Traefik Ingress | 100m | 128 Mi | 默认 Ingress Controller |
| metrics-server | 100m | 64 Mi | 资源监控 |
| local-path-provisioner | 50m | 32 Mi | 默认存储 |
| kube-proxy (kube-router) | 50m | 32 Mi | 网络代理 |
| **Server 节点合计** | **~900m** | **~1.2 Gi** | **系统开销，不含业务负载** |

### Agent 节点（每台）

| 组件 | CPU（最低） | RAM（最低） | 说明 |
|------|------------|------------|------|
| k3s agent 进程 | 100m | 256 Mi | kubelet + kube-proxy |
| containerd | 100m | 128 Mi | 容器运行时 |
| **Agent 节点合计** | **~200m** | **~384 Mi** | **系统开销，不含业务负载** |

## 整体集群最低配置建议

| 节点角色 | 数量 | 最低 CPU | 最低 RAM | 推荐 CPU | 推荐 RAM | 磁盘 |
|---------|------|---------|---------|---------|---------|------|
| Server | 3 | 2 vCPU | 4 GB | 4 vCPU | 8 GB | 50 GB SSD |
| Agent | 2 | 2 vCPU | 4 GB | 4 vCPU | 8 GB | 50 GB SSD |
| **集群总计** | **5** | **10 vCPU** | **20 GB** | **20 vCPU** | **40 GB** | **250 GB SSD** |

---

## 关键说明

1. **Server 节点 3 台**：etcd 要求奇数节点才能形成 quorum，3 台可容忍 1 台故障
2. **磁盘 I/O**：etcd 对磁盘延迟敏感，必须用 SSD，HDD 会导致集群不稳定
3. **PostgreSQL**：如果数据量大或并发高，建议独立部署在 Server 节点外，或者给 PG 节点分配更多资源（4 vCPU / 8 GB+）
4. **预留 20-30% 余量**：K8s 调度需要 buffer，节点资源打满会导致 Pod 驱逐
5. **OS 开销**：每台机器系统本身约占 0.5 vCPU + 512 Mi RAM，上表未含此部分

> 如果预算有限，最低可以用 **2C4G × 5** 起步，但生产环境建议 **4C8G × 5**。
