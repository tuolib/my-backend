# Single Server 部署架构

> 单机 Docker Compose 部署方案，适用于学习/演示环境。最低 1 核 1G + swap。

---

## 架构总览

```
                    Internet
                       │
                       ▼
              ┌────────────────┐
              │   Nginx :80/443│  SSL 终结 + 反向代理
              └───────┬────────┘
                      │
              ┌───────▼────────┐
              │ API Gateway    │  唯一入口 :3000
              │                │  路由分发 + 中间件链
              └───────┬────────┘
                      │ HTTP（Docker 内部网络）
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
  ┌──────────┐ ┌──────────┐ ┌──────────┐
  │  User    │ │ Product  │ │  Order   │
  │  :3001   │ │  :3002   │ │  :3004   │
  └────┬─────┘ └────┬─────┘ └──┬───┬──┘
       │            │           │   │
       │       ┌────▼─────┐    │   │
       │       │  Cart    │◄───┘   │
       │       │  :3003   │        │
       │       └────┬─────┘        │
       │            │              │
       ▼            ▼              ▼
  ┌──────────────────────────────────┐
  │         data_net (internal)      │
  │  ┌────────────┐ ┌─────────────┐ │
  │  │ PostgreSQL │ │    Redis    │ │
  │  │   :5432    │ │    :6379    │ │
  │  └────────────┘ └─────────────┘ │
  └──────────────────────────────────┘
```

## 服务清单（9 个）

| 服务 | 镜像 | 作用 | 网络 |
|------|------|------|------|
| **nginx** | nginx:1.27-alpine | SSL 终结、反向代理、ACME challenge | app_net |
| **certbot** | certbot/certbot | Let's Encrypt 自动签发/续签 | app_net |
| **api-gateway** | ecom-api-gateway | 请求路由、中间件（认证/限流/幂等） | app_net + data_net |
| **user-service** | ecom-user-service | 用户注册、登录、JWT 签发 | app_net + data_net |
| **product-service** | ecom-product-service | 商品 CRUD、库存管理 | app_net + data_net |
| **cart-service** | ecom-cart-service | 购物车（Redis 存储） | app_net + data_net |
| **order-service** | ecom-order-service | 下单、支付、库存扣减 | app_net + data_net |
| **postgres** | postgres:16-alpine | 主数据库 | data_net |
| **redis** | redis:7-alpine | 缓存/购物车/分布式锁/延迟队列 | data_net |

---

## 网络隔离

| 网络 | 类型 | 用途 | 可访问服务 |
|------|------|------|-----------|
| **app_net** | bridge | 应用层通信 | nginx、certbot、5 个应用服务 |
| **data_net** | bridge (internal) | 数据层通信，**外部不可达** | postgres、redis、5 个应用服务 |

- Nginx 只在 app_net，不能直连数据库
- PostgreSQL/Redis 只在 data_net，不暴露端口到宿主机
- 应用服务横跨两个网络，既能被 Nginx 代理，也能访问数据库

## 与 Swarm 方案对比

| | Swarm（5 台） | Single Server（1 台） |
|---|---|---|
| PostgreSQL | 2x Patroni + 3x etcd（自动故障转移） | 1x 直连 |
| Redis | 主从 + 3x Sentinel + HAProxy | 1x 直连 |
| Nginx | ingress mesh 多节点入口 | 单实例 |
| SSL | Docker Secrets 轮换 | 文件卷挂载 |
| 应用服务 | 5 个（跨节点调度） | 5 个 |
| 监控 | node-exporter + cAdvisor | 无 |
| **总服务数** | **20** | **9** |
| **去掉的组件** | — | etcd、Patroni、Sentinel、HAProxy、监控 |

## 服务器要求

| 资源 | 最低 | 推荐 |
|------|------|------|
| CPU | 1 核 | 2 核 |
| 内存 | 1 GB + 1G swap | 2 GB |
| 磁盘 | 20 GB SSD | 40 GB SSD |

内存估算（空闲状态）：

```
OS + Docker     ~350 MB
PostgreSQL      ~100 MB
Redis           ~30 MB
Bun 应用 x5     ~350 MB
Nginx + Certbot ~10 MB
─────────────────────────
合计             ~840 MB
```

> 1G 内存需配置 swap 防止启动阶段 OOM，运行起来后空闲占用低于 1G。

## 服务器文件结构

```
/opt/ecom/                        # 部署根目录
├── .env                          # 环境变量（CI 自动生成，chmod 600）
├── docker-compose.prod.yml       # Compose 编排文件
├── nginx.conf                    # Nginx 配置
├── certbot-renew.sh              # SSL 签发/续签脚本
├── postgresql.conf               # PG 调优配置（适配 2G 以下）
├── redis.conf                    # Redis 配置（256MB 内存上限）
├── init.sql                      # PG schema 初始化
├── ops.sh                        # 运维工具
└── ssl/                          # SSL 证书
    ├── fullchain.pem             #   自签/Let's Encrypt 证书
    └── privkey.pem               #   私钥
```

## 请求链路

```
用户请求 → DNS → 服务器 :443
  → Nginx（SSL 终结）
    → API Gateway :3000（认证/限流/幂等/路由）
      → user-service :3001      ← /api/v1/auth/*, /api/v1/users/*
      → product-service :3002   ← /api/v1/products/*, /api/v1/skus/*
      → cart-service :3003      ← /api/v1/cart/*
      → order-service :3004     ← /api/v1/orders/*
        → PostgreSQL :5432      ← SQL 读写
        → Redis :6379           ← 缓存/锁/队列
```

## 数据持久化

| Docker Volume | 服务 | 数据内容 |
|---------------|------|---------|
| `pg_data` | PostgreSQL | 数据库文件 |
| `redis_data` | Redis | AOF 持久化文件 |
| `certbot_conf` | Certbot | Let's Encrypt 证书 |
| `certbot_webroot` | Certbot + Nginx | ACME challenge 文件 |

> `docker compose down` 不会删除 volume，数据安全。
> `docker compose down -v` 会删除 volume，**数据全部丢失**。

## SSL 证书流程

```
1. init-server.sh 生成自签证书 → /opt/ecom/ssl/
2. Nginx 启动，使用自签证书监听 :443
3. Certbot 启动：
   a. 等待 Nginx ready
   b. 通过 webroot 模式申请 Let's Encrypt 证书
      （Nginx 代理 /.well-known/acme-challenge/ → certbot 写入的文件）
   c. 证书写入 certbot_conf volume
4. 每 12 小时自动检查续签
```

## 启动顺序（depends_on）

```
postgres (healthy) ──┐
                     ├→ api-gateway (healthy) → nginx
redis (healthy) ─────┤
                     ├→ user-service
                     ├→ product-service
                     ├→ cart-service
                     └→ order-service
```

所有应用服务等 PG + Redis 健康后才启动，Nginx 等 API Gateway 健康后才启动。
